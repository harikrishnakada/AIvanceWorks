# QA, Security & Compliance

## 10. QA, SECURITY & COMPLIANCE
### Quality Assurance
- Automated testing (unit, integration, E2E)
- Test-Driven Development (TDD) practices where applicable
- Performance and load testing
- Accessibility testing

### Security & Compliance
- SOC 2 alignment, HIPAA support for relevant clients
- Regular penetration testing and vulnerability scanning
- Role-based access control (RBAC), least-privilege design
- Data encryption at-rest and in-transit
- Security code reviews and dependency vulnerability scanning
