Pilot build scope for the new unified services/solutions design system is five detail pages:

- `/services/product-discovery`
- `/services/mvp-development`
- `/solutions/patient-portals`
- `/solutions/insurance-portals`
- `/solutions/e-commerce-websites` (added 2026-04-10 via e-commerce migration feature)

**Why:** User wanted to validate the shared-skeleton + two-variants approach (Option B + uniform palette) on a small, meaningful pilot before rolling out to the rest of the solutions and services in the menu. The original 4 pages (2 services + 2 solutions) proved both template variants under the same design system. E-commerce was then migrated from the legacy fallback path into the pilot architecture, expanding the pilot to 5.

**How to apply:** When scoping implementation plans, use the five pilot pages as reference implementations. Remaining pages get created by filling out data files and (optionally) adding a signature component, using the existing pages as templates.
